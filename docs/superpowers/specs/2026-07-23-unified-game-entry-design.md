# 统一游戏入口设计

## 背景

项目当前有两个容易被理解为“首页”的地址：

- `/`：三国霸业的启动、设置页面，同时包含其他游戏链接。
- `/bbk-games/index.html`：步步高电子词典模拟器及 ROM 选择页面。

两个页面的视觉和导航体系互不一致。根页面还混合了三国霸业专属设置与全站游戏导航，页面层级不清晰。

## 目标

1. 将 `/` 改造成唯一的游戏大厅和推荐入口。
2. 保留 `/bbk-games/index.html` 为步步高电子词典模拟器的直达地址。
3. 在新的游戏大厅中保留三国霸业现有的启动、版本选择、存档传送和显示设置。
4. 建立适合经典电子词典游戏的统一视觉，并兼顾手机和桌面端。
5. 不改变各游戏内部逻辑、ROM 文件或已有存档格式。

## 非目标

- 不重写 `/bbk-games/index.html` 中的 React 模拟器。
- 不重构三国霸业、魔塔或 RPG 游戏引擎。
- 不增加账号、云存档、搜索、收藏或最近游玩等新功能。
- 不更改现有 `localStorage` 键名。

## 路由与入口

| 页面 | 用途 | 行为 |
| --- | --- | --- |
| `/` | 统一游戏大厅 | 展示全部游戏入口与三国霸业设置 |
| `/bbk-games/index.html` | 步步高电子词典模拟器 | 保持直达，不重定向 |
| `/mt.html` | 魔塔 | 从游戏大厅直接进入 |
| `/fm/index.html` | RPG 游戏列表 | 从游戏大厅直接进入 |
| `/choose.html` | 三国霸业版本选择 | 从设置抽屉进入 |
| `/get-sav.html` | 三国霸业存档传送 | 从设置抽屉进入 |

所有现有目标页面及其内部导航保持不变。

## 首页信息架构

首页采用单页游戏大厅结构：

1. 顶部品牌区：显示 `BBK · 经典游戏` 和状态指示灯。
2. LCD 标题区：显示“请选择游戏”及简短操作提示。
3. 游戏列表：展示四张游戏卡片。
   - 三国霸业
   - 步步高电子词典游戏
   - 魔塔
   - RPG 游戏
4. 三国霸业卡片包含两个明确入口：
   - 主操作：使用现有启动逻辑进入游戏。
   - 设置操作：打开三国霸业设置抽屉。
5. 页脚：用简短文字说明项目为经典电子词典游戏的网页移植合集。

手机端使用单列游戏列表；桌面端使用两列布局，并限制主体最大宽度，避免内容在宽屏中过度拉伸。

## 三国霸业设置抽屉

设置抽屉保留以下功能：

- 选择版本：进入 `/choose.html`。
- 传送存档：进入 `/get-sav.html`。
- 操作模式：
  - 横屏与触控
  - 竖屏与键盘
  - 横屏与手势
- 分辨率：
  - 4988
  - 加大
- 像素锐化：
  - 禁用
  - 启用

设置值继续写入现有键：

- `baye/mpage`
- `baye/resolution`
- `baye/clearmode`

抽屉交互规则：

- 手机端从底部弹出；桌面端显示为居中对话框。
- 点击设置按钮打开。
- 点击关闭按钮、遮罩或按 `Esc` 关闭。
- 打开时将焦点移入抽屉，关闭后将焦点还给触发按钮。
- 打开时禁止背景滚动。
- 使用原生 `select`，以保留移动端系统选择体验。

## 视觉设计

采用“电子词典拟物”方向，但控制装饰强度，确保页面仍像现代游戏大厅：

- 页面底色使用浅灰绿，模拟电子词典机身。
- LCD 区使用低饱和黄绿色、深灰边框和轻微内阴影。
- 游戏编号和主要按钮模拟实体塑料按键。
- 使用等宽字体呈现 LCD 文案；普通说明使用系统无衬线字体。
- 通过少量扫描线、像素角标和状态灯建立怀旧感。
- 交互反馈使用轻微下压和阴影变化，不依赖复杂动画。
- 在 `prefers-reduced-motion` 下关闭非必要过渡。
- 文字与背景保持足够对比度，焦点样式清晰可见。

不新增图片素材。游戏卡片使用文字、编号和 CSS 图形构成，以减少加载开销并保持风格一致。

## 文件与组件边界

### `index.html`

只负责首页语义结构和内容：

- 品牌区
- LCD 标题区
- 游戏卡片
- 三国霸业设置抽屉
- 无脚本回退信息

### `css/portal.css`

只负责统一入口的视觉和响应式布局：

- 主题变量
- 页面布局
- LCD 与实体按键视觉
- 游戏卡片
- 抽屉与遮罩
- 桌面/手机断点
- 减少动画和焦点状态

首页不再依赖为游戏画面与虚拟按键设计的 `css/baye.css`。

### `js/portal.js`

只负责入口页交互：

- 读取和保存三项现有设置
- 打开和关闭设置抽屉
- 管理焦点与背景滚动
- 响应 `Esc`

三国霸业启动继续调用现有 `js/lcd.js` 中的 `redirect()`，避免复制设备识别、页面选择和游戏版本读取逻辑。首页继续加载 jQuery 和 `js/lcd.js`，因为现有启动函数依赖它们。

## 数据流

1. 页面载入时，`portal.js` 从 `localStorage` 读取三项设置。
2. 若键不存在或读取失败，对应控件使用值 `0`。
3. 用户修改设置时立即写回同一个 `localStorage` 键。
4. 用户点击“进入游戏”时调用现有 `redirect()`：
   - 桌面端进入 `pc.html`。
   - 手机端根据 `baye/mpage` 选择对应页面。
   - 游戏版本继续由现有 `getLibName()` 获取。
5. 其他游戏卡片使用普通链接直接导航，不经过 JavaScript。

## 异常与回退

- `localStorage` 读取或写入失败时不阻止页面使用，设置回退到默认值。
- JavaScript 不可用时，步步高电子词典、魔塔、RPG 游戏、版本选择和存档传送仍通过普通链接可访问。
- 三国霸业主操作提供指向 `pc.html` 的链接作为基础回退；JavaScript 可用时拦截该链接并调用现有 `redirect()`，从而保留移动端适配逻辑。
- 不存在的旧设置值统一显示为默认选项，但不主动覆盖存储值，直到用户进行修改。

## 验证方案

### 自动检查

- 首页无缺失的本地 CSS、JavaScript 或图片资源。
- 四个游戏入口的 `href` 指向预期地址。
- 三项设置使用正确的 `localStorage` 键。
- 页面存在唯一的主标题和合理的语义结构。

### 浏览器验证

- 390×844 手机视口：单列卡片、底部抽屉、无水平滚动。
- 1440×900 桌面视口：双列卡片、居中对话框、内容宽度合理。
- 抽屉可通过按钮、遮罩和 `Esc` 关闭。
- 抽屉打开与关闭时焦点正确转移和恢复。
- 修改三项设置并刷新后，值仍然保留。
- 桌面端三国霸业进入 `pc.html`。
- `/bbk-games/index.html` 仍能直接加载模拟器和游戏选择。
- `/mt.html` 与 `/fm/index.html` 仍可从大厅进入。

## 验收标准

- 用户访问 `/` 时看到统一、完整且风格一致的游戏大厅。
- 首页不再直接铺开三国霸业设置。
- 四类游戏均可在一次点击内进入或开始启动。
- 三国霸业的版本、存档和显示设置没有功能缺失。
- `/bbk-games/index.html` 保持原功能和直达能力。
- 手机和桌面页面均无明显溢出、遮挡或不可操作控件。
