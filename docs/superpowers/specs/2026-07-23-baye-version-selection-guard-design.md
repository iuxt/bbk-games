# 三国霸业启动版本选择保护设计

## 背景

三国霸业通过 `js/lcd.js` 中的 `redirect()` 从游戏大厅选择桌面端或移动端页面，并在游戏页通过 `bayeMain()` 读取 `localStorage` 中的 `baye/libpath` 加载版本。当前没有选择版本时，游戏页只弹出“没有选择版本”，用户需要手动返回并找到版本选择入口。

## 目标

- 从游戏大厅启动三国霸业时，如果尚未选择版本，直接进入 `choose.html`。
- 直接访问 `pc.html`、`m.html`、`m-ktouch.html` 或 `m-ges.html` 时，如果尚未选择版本，也直接进入 `choose.html`。
- 已选择版本时，保留现有桌面端、移动端识别以及游戏加载行为。
- 选择一个版本后立即按现有设备识别规则启动游戏。

## 非目标

- 不改变版本列表、版本说明或版本文件。
- 不改变现有 `localStorage` 键名。
- 不改变游戏画面、控制模式或设备识别规则。
- 不新增默认版本。

## 方案

版本选择保护集中在 `js/lcd.js` 的公共启动流程中。

1. 增加一个读取当前版本路径的小函数，以 `baye/libpath` 是否存在作为“已选择版本”的唯一判断。
2. `redirect()` 在设备识别和页面拼接前检查版本路径。路径不存在时设置 `window.location.href` 为 `choose.html` 并立即返回。
3. `bayeMain()` 在实际加载版本前执行同样的检查，作为直接访问游戏页面时的兜底。路径不存在时跳转到 `choose.html` 并立即返回，不再弹窗。
4. 把清理旧版本数据与跳转拆开。`chooseLib()` 先清理旧版本数据，再保存新的版本名称和路径，最后调用 `redirect()`。这样新增的启动保护不会在新版本写入前误判。
5. `clearLib()` 继续清除当前版本，但清除后由 `redirect()` 自动进入版本选择页。

## 数据流

### 从游戏大厅启动

1. 用户点击“三国霸业”。
2. 游戏大厅调用现有 `redirect()`。
3. `redirect()` 读取 `baye/libpath`。
4. 未选择版本时进入 `choose.html`；已选择版本时继续进入设备对应的游戏页。

### 直接访问游戏页

1. 游戏页初始化并调用 `bayeMain()`。
2. `bayeMain()` 读取 `baye/libpath`。
3. 未选择版本时进入 `choose.html`；已选择版本时加载对应版本并启动游戏。

### 选择版本

1. 用户在 `choose.html` 点击一个版本。
2. 清理旧版本文件缓存、名称和路径。
3. 写入新版本名称和路径。
4. 调用 `redirect()`，进入设备对应的游戏页。

## 异常与兼容性

- 空字符串和缺失的 `baye/libpath` 都视为未选择版本。
- 版本路径存在时不额外验证网络文件是否可访问，保持当前加载失败行为不变。
- 页面仍使用相对地址 `choose.html`，与当前所有三国霸业入口位于同一目录的结构一致。
- 不修改游戏大厅普通链接的 `href`，保留现有无 JavaScript 回退行为。

## 测试

新增针对 `js/lcd.js` 启动行为的自动化测试：

- `redirect()` 在没有版本路径时跳转 `choose.html`。
- `redirect()` 在已有版本路径时继续执行现有桌面端或移动端跳转。
- `bayeMain()` 在没有版本路径时跳转 `choose.html`，且不调用版本加载器。
- `bayeMain()` 在已有版本路径时继续加载版本。
- `chooseLib()` 在写入新版本后再启动，不会被版本选择保护送回 `choose.html`。

运行现有游戏大厅测试，确认入口、设置和回退链接没有回归。
